## Why use this over Claude Code or Codex?

Closed hosted tools can send prompts, code, and engineering context outside your trust boundary. The ORANGE Line gives your team frontier-level coding-agent intelligence while keeping inference traffic, runtime operations, and deployment policy inside infrastructure you control.

See [Compliance](/on-prem/trust-center/compliance) for the data and IP boundary.

## Why use Subconscious instead of hosting the models and managing GPUs ourselves?

You can rent GPUs and host open models yourself, but the hard part is serving coding-agent workloads efficiently and reliably. Subconscious helps manage the runtime, gateway, deployment workflow, admin surfaces, upgrades, and support.

The ORANGE Line is designed to serve teams of engineers with roughly half the GPUs compared with off-the-shelf inference runtimes like vLLM.

See [How it works](/on-prem/how-it-works) and [Configurations](/on-prem/deployment-models/configurations).

## Will this compromise our SOC 2 posture?

No. The customer-hosted deployment is designed so the runtime plane runs in your cloud account or controlled environment. You retain control over deployment, access, monitoring, networking, data, and change management.

See [Compliance](/on-prem/trust-center/compliance).

## Where does the runtime plane run?

The runtime plane runs in the customer's cloud or controlled infrastructure. Depending on your configuration, GPU workers may run in the same cloud, another customer-controlled environment, or a specialized GPU provider behind the gateway.

See [How it works](/on-prem/how-it-works) and [Configurations](/on-prem/deployment-models/configurations).

## What data does Subconscious access?

By default, production prompts, completions, source code, API keys, runtime logs, and operational data should remain in the customer environment. Customers may choose to share selected logs, screenshots, metrics, or traces for support.

See [Compliance](/on-prem/trust-center/compliance).

## How are upgrades and patches delivered?

Subconscious delivers releases, patches, and release metadata through the software distribution plane. Customers can approve updates through their own change-management process or choose an assisted update workflow.

See [Distribution platform](/on-prem/deployment-models/distribution-platform) and [Customer success](/on-prem/customer-journey/customer-success).

## Can we control when updates are deployed?

Yes. Production updates are intended to be customer-approved and deployed according to your policy, maintenance windows, and change-management requirements.

See [Methods](/on-prem/deployment-models/methods) and [Compliance](/on-prem/trust-center/compliance).

## Do you support vulnerability scanning?

Subconscious can provide or work toward providing release evidence such as vulnerability scan results, SBOMs, image digests, checksums, and release notes. Customers can also scan release artifacts and deployed components with their own tools.

See [Compliance](/on-prem/trust-center/compliance) and [Distribution platform](/on-prem/deployment-models/distribution-platform).

## What clouds and Kubernetes environments are supported?

The deployment can be planned for AWS, GCP, Azure, NeoCloud or dedicated GPU providers, and customer-controlled Kubernetes environments. Exact support depends on your cloud, networking, GPU placement, registry, and security requirements.

See [Configurations](/on-prem/deployment-models/configurations).

## Do we need to bring our own GPUs?

Not always. Customers can provide approved GPU capacity, use cloud GPU resources, or route through specialized GPU providers depending on the deployment design. We can also help you source GPU capacity through our network of partnerships.

See [Configurations](/on-prem/deployment-models/configurations).

## If a new model comes out, can we deploy it?

Yes, as long as the model architecture is supported by the runtime and enough GPU capacity is available. If existing capacity is already allocated, we may need to add GPU resources or replace an existing route before deploying the new model.

See [Customer success](/on-prem/customer-journey/customer-success).

## Which coding agents are supported?

The gateway exposes OpenAI- and Anthropic-compatible endpoints, so teams can connect common coding agents (Claude Code, Cursor, Codex, OpenCode, Pi, etc.) and internal tools that support those API shapes.

See [Gateway setup](/on-prem/deployment-models/gateway-setup).

## What happens if the deployment has an incident?

The customer owns incident response for the runtime plane in its environment. Subconscious supports investigation and remediation when requested by the customer, using customer-approved access or diagnostic sharing.

See [Customer success](/on-prem/customer-journey/customer-success) and [Compliance](/on-prem/trust-center/compliance).
