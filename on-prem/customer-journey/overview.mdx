This guide explains the path from first evaluation to production use of The ORANGE Line.

## Typical phases

- **[Evaluation](/on-prem/customer-journey/evaluation)**: Align on use cases, security posture, ROI, expected usage, and proof points.
- **[Onboarding](/on-prem/customer-journey/onboarding)**: Plan deployment, stand up The ORANGE Line, configure the API Gateway, and connect pilot users.
- **[Customer success](/on-prem/customer-journey/customer-success)**: Keep the support channel open for upgrades, patches, performance reviews, and rollout guidance.

## Rough timeline

Most assisted deployments can move from kickoff to first pilot use in **1-2 weeks** when customer technical resources are available.

- Security review can run in parallel with deployment planning.
- Initial configuration usually takes less than a day of focused technical time from the customer champion.
- Strict change management, private registries, air-gapped workflows, new GPU procurement, or custom networking can extend the timeline.

## Champions and stakeholders

The best technical champion is usually close enough to infrastructure to unblock deployment and close enough to the engineering organization to understand coding-agent adoption.

Common champions and stakeholders include:

- **Platform engineering lead**: Owns Kubernetes, deployment standards, internal developer platforms, or ORANGE Line operations.
- **Infrastructure owner**: Helps with cloud account, region, networking, DNS, TLS, load balancers, GPU capacity, and observability.
- **Security or compliance reviewer**: Reviews the data boundary, support access model, software supply-chain evidence, and SOC 2 vendor posture.
- **Developer productivity lead**: Connects the deployment to coding agents, rollout plans, and engineering workflow improvements.
- **AI tooling owner**: Defines model quality expectations, pilot workflows, and compatibility with existing agent tools.
- **Engineering sponsor**: Confirms business value, pilot success criteria, and expansion goals.

## What to prepare

Before onboarding, it helps to identify:

- Target cloud, region, Kubernetes environment, and deployment boundary.
- GPU capacity plan for the Inference Runtime, whether colocated with the API Gateway or provided by a specialized GPU environment.
- Preferred deployment method: [assisted self-managed or traditional self-hosted](/on-prem/deployments/methods).
- Security review owners and required evidence.
- Initial pilot users, coding agents, model routes, and usage expectations.
- Admin users responsible for user management, monitoring, and support escalation.

For detailed deployment and configuration instructions, use the [Deployments](/on-prem/deployments/methods) section.
